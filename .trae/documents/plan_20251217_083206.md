## 修复Cold Start Fetch SQLITE_ERROR错误

### 核心问题
SQLITE_ERROR: incomplete input错误是由userService.js中getAllUsers方法的SQL查询语句语法错误导致的。

### 错误分析

**文件**: `src/main/services/userService.js`

**错误代码**: 
```javascript
async getAllUsers() {
  const sql = 'SELECT * FROM Mysqlks WHERE' // 语法错误！WHERE子句没有条件
  const rows = await this._fetchAll(sql, [])
  // ...
}
```

**调用方式**: 
```javascript
const users = await this.usersSvc.getAllUsers(false) // 传入了false参数，但方法没有定义参数
```

### 修复方案

#### 1. 修复SQL语句
- 去掉多余的`WHERE`子句，改为完整的SQL查询语句

#### 2. 兼容现有调用
- 修改getAllUsers方法，使其接受并忽略参数（为了兼容现有调用）

### 具体修改

**文件**: `src/main/services/userService.js`

**修改内容**:
```javascript
// 修复前
async getAllUsers() {
  const sql = 'SELECT * FROM Mysqlks WHERE' // 语法错误
  // ...
}

// 修复后
async getAllUsers(includeInactive = false) {
  const sql = 'SELECT * FROM Mysqlks' // 修复SQL语句，去掉多余的WHERE子句
  // ...
}
```

### 修复效果

1. 修复了SQL语法错误，消除了SQLITE_ERROR: incomplete input错误
2. 保持了对现有调用的兼容性
3. 使getAllUsers方法能够正确获取所有用户数据
4. 确保Cold Start Fetch能够正常完成
