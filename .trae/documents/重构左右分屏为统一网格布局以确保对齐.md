## 布局选择
- 采用 CSS Grid 架构作为顶层布局（对齐最简单可靠）：
  - 容器：`display: grid; grid-template-columns: var(--sidebar-width) 1fr; gap: var(--gap); height: 100vh; padding: var(--gap); align-items: stretch; box-sizing: border-box`
  - 统一用 `ElCard` 包裹左右两侧（右侧内容也放入卡片），两侧设置 `height: 100%`，对齐无偏差
  - 用 CSS 变量统一侧边栏宽度与间距（易调整、全局生效）：`--sidebar-width: 300px`（可在 280–320 调整）；折叠宽度 `64px`

## 左侧 Sidebar（ElCard）
- 固定宽度 `var(--sidebar-width)`；折叠时宽度变 `64px`
- 结构：`ElCard > Header(标题+折叠按钮) + ElMenu(导航) + Footer`
- 滚动处理：内部使用 `ElScrollbar`，父级 `height: 100%`，侧边卡片 `height: 100%`
- 响应式：`@media (max-width: 768px)` 自动折叠；保留展开按钮

## 右侧内容区（ElCard）
- 右侧同样用 `ElCard` 包裹（与左侧一致的半径、阴影、内边距），保证外观对齐
- 内容卡片内：顶部 `HeaderActions`、中部 `DataTable`（保持现有计时&IPC数据订阅）
- 滚动：右侧卡片内的 `ElScrollbar`，避免外层滚动破坏高度对齐

## 状态与通信
- 保留 Pinia（`uiStore`）维护折叠状态 `collapsed` 与当前菜单项 `activeKey`
- 路由切换与内容解耦：点击侧边菜单只更新 `uiStore.activeKey` 并 `router.push(route)`
- `keep-alive` 缓存右侧组件，保证计时器与数据持久

## 代码改动
1) 新增 `ShellLayout.vue`（顶层网格）：
- 模板：两列网格，左 `SidebarCard`，右 `ContentCard`
- 暴露插槽：`sidebar-header/sidebar-footer`、`content-header/content-body`
- 右侧内容卡片将现有 `MainView` 作为默认内容（仍通过路由加载）

2) 样式统一：
- 在 `:root` 或 `App.vue` 设置 `--sidebar-width` 与 `--gap`
- 左右 `ElCard` 的 `height: 100%`；内部使用 `ElScrollbar`

3) App集成：
- `App.vue` 使用 `ShellLayout`；`router-view` 放入右侧内容卡片的 `content-body` 插槽

## 验证
- 左右高度对齐：窗口大小变化仍对齐
- 表格计时：切换菜单不影响计时与数据；返回仍保持现状
- 响应式：768px 以下侧边栏自动折叠，右侧内容占满
